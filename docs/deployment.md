# Deployment

## GitHub Pages

The workflow publishes the `site/` directory generated by MkDocs.

Configure the repository in GitHub with:

- Actions enabled.
- Pages source set to GitHub Actions.
- Workflow permissions set to read and write.
- Default branch named `main`.

## Readiness Check

Run this before the first push:

```powershell
./scripts/check-github-readiness.ps1
```

Use strict mode in CI or release checks:

```powershell
./scripts/check-github-readiness.ps1 -FailOnWarnings
```

## Branch Rules

- Pull requests validate documentation only.
- Pushes to feature branches generate and validate documentation.
- Pushes to `main` generate, validate, commit changes and publish.

## First Push

If the repository is not initialized yet:

```powershell
git init
git branch -M main
git add .
git commit -m "chore: add automated documentation"
git remote add origin https://github.com/OWNER/REPOSITORY.git
git push -u origin main
```

Replace `OWNER/REPOSITORY` with the real GitHub repository.

## Auto-Commit

The workflow uses this commit message for generated documentation:

```text
chore(docs): update generated documentation [skip ci]
```

The `[skip ci]` suffix avoids unnecessary workflow loops.

## Workflow Permissions

The workflow uses:

- `contents: write` for generated documentation commits.
- `pages: write` for GitHub Pages deployment.
- `id-token: write` for the Pages deployment token.
